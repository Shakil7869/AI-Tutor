1. Create Firebase Project

Steps:

  1. Go to Firebase Console
  2. Click "Create a project"
  3. Project name: ai-tutor-nctb
  4. Enable Google Analytics (optional)
  5. Choose your country/region
  6. Click "Create project"

2. Enable Firebase Services

Firebase Storage:

  1. In your Firebase project, go to Storage
  2. Click "Get started"
  3. Start in test mode (we'll secure it later)
  4. Choose your storage location

Firestore Database:

  1. Go to Firestore Database
  2. Click "Create database"
  3. Start in test mode
  4. Choose your region (same as Storage)

3. Download Service Account Key

For Python Service:

  1. Go to Project Settings (gear icon)
  2. Go to Service accounts tab
  3. Click "Generate new private key"
  4. Download the JSON file
  5. Rename it to firebase_config.json
  6. Place it in: pdf_management_service/config/firebase_config.json
Security Note: Keep this file secure! It provides admin access to your Firebase project.

4. Update Configuration

Update Storage Bucket Name:

In pdf_manager_firebase.py, find line 53 and update:

'storageBucket': 'your-project-id.appspot.com' # Replace with your actual bucket

Replace with your actual project ID from Firebase Console.

5. Restart the Service

Commands:

# Stop current service (Ctrl+C) # Then restart with Firebase support: cd pdf_management_service python pdf_manager_firebase.py

6. Security Rules (Important!)

Firestore Rules:

Go to Firestore ‚Üí Rules and replace with:

rules_version = '2'; service cloud.firestore { match /databases/{database}/documents { match /nctb_chapters/{document} { allow read: if true; allow write: if request.auth != null; } match /nctb_pdfs/{document} { allow read: if true; allow write: if request.auth != null; } } }

Storage Rules:

Go to Storage ‚Üí Rules and replace with:

rules_version = '2'; service firebase.storage { match /b/{bucket}/o { match /textbooks/{allPaths=**} { allow read: if true; allow write: if request.auth != null; } match /chapters/{allPaths=**} { allow read: if true; allow write: if request.auth != null; } } }

  • Automatic Sync: Real-time updates
  • Scalability: Handle multiple users
  • Offline Support: Cached data works offline

  3. Configure chapters - they will be saved to Firestore
  4. Check Firebase Console to see your data
